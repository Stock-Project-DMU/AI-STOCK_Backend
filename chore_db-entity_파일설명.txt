--------------------------------------------------------------------
domain/account  (accounts 테이블)
--------------------------------------------------------------------
entity/Account.java
  accounts 테이블 매핑. @Version으로 낙관적 락 적용(지정가 동시 체결
  대비), balance/frozen_balance/base_balance 보유. User와 N:1.

repository/AccountRepository.java
  user_id로 계좌 조회(JpaRepository 상속). 주문 체결 시 잔고 검증에 사용.

--------------------------------------------------------------------
domain/order  (holdings, orders 테이블)
--------------------------------------------------------------------
entity/Holding.java
  holdings 테이블 매핑. 계좌별 보유 종목/수량/평균단가.

entity/Order.java
  orders 테이블 매핑. 주문 요청가/체결가/수량/상태 보유.

entity/OrderStatus.java   - PENDING / EXECUTED / CANCELLED
entity/OrderType.java     - BUY / SELL
entity/PriceType.java     - LIMIT / MARKET

repository/HoldingRepository.java
  account_id + stock_code로 보유 종목 조회. 매도 시 잔여수량 검증에 사용.

repository/OrderRepository.java
  지정가 미체결 주문 조회 등. 서버 시작 시 PENDING 주문을 Redis에
  재적재하는 로직, 마이페이지 주문내역 조회에서 사용.

--------------------------------------------------------------------
domain/stock  (watchlist, recent_viewed 테이블)
--------------------------------------------------------------------
entity/Watchlist.java      - watchlist 테이블 매핑. 관심종목.
entity/RecentViewed.java   - recent_viewed 테이블 매핑. 최근 본 종목
                              (재조회 시 viewed_at UPSERT).

repository/WatchlistRepository.java     - user_id로 관심종목 CRUD.
repository/RecentViewedRepository.java  - user_id+stock_code UPSERT 조회.

--------------------------------------------------------------------
domain/ai  (ai_planning_sessions, ai_planning_messages, simulations)
--------------------------------------------------------------------
entity/AiPlanningSession.java
  ai_planning_sessions 테이블 매핑. AI 재무설계 대화 세션.

entity/SessionStatus.java  - ACTIVE / CLOSED

entity/AiPlanningMessage.java
  ai_planning_messages 테이블 매핑. Gemini API는 stateless이므로
  대화 이력을 직접 저장해 재접속 시 복원 + 프롬프트 구성에 사용.

entity/MessageRole.java    - USER / AI

entity/Simulation.java
  simulations 테이블 매핑. 목표 도달 시뮬레이션 결과
  (Best/Base/Worst 시나리오, DART/Tavily 원본 JSON 포함).

repository/AiPlanningSessionRepository.java  - 세션 목록/상태 조회.
repository/AiPlanningMessageRepository.java  - 세션별 메시지 이력 조회.
repository/SimulationRepository.java         - user_id로 시뮬레이션 이력 조회.

--------------------------------------------------------------------
domain/notification  (notifications 테이블)
--------------------------------------------------------------------
entity/Notification.java       - notifications 테이블 매핑.
entity/NotificationType.java   - SYSTEM / ORDER / AI / SIMULATION

repository/NotificationRepository.java
  user_id + is_read로 안읽은 알림 조회. STOMP 유니캐스팅과 함께 사용.

--------------------------------------------------------------------
domain/user  (social_accounts, investment_profile 테이블)
  ※ User / Role은 chore/db-entity가 아니라 feature/jwt-common이 구현
--------------------------------------------------------------------
entity/SocialAccount.java
  social_accounts 테이블 매핑. User와 N:1, 카카오/네이버/구글 다중
  연동 지원.

entity/SocialProvider.java  - KAKAO / NAVER / GOOGLE

entity/InvestmentProfile.java
  investment_profile 테이블 매핑. User와 1:1 (설문 결과, 1인 1건).

entity/InvestmentLevel.java  - BEGINNER / INTERMEDIATE / EXPERT

repository/SocialAccountRepository.java
  provider + provider_id로 소셜 계정 조회. 소셜 로그인 시 사용.

repository/InvestmentProfileRepository.java
  user_id로 투자성향 조회. 마이페이지/AI 상담에서 사용.

--------------------------------------------------------------------
global/config
--------------------------------------------------------------------
JpaConfig.java
  @Configuration + @EnableJpaAuditing 추가. 모든 엔티티의
  created_at/updated_at 자동 처리를 활성화.
